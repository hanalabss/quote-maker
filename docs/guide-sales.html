정보 수정 / 로그아웃 — 프로필 수정 및 로그아웃
2
견적 요청 — 새로운 견적을 요청합니다. 행사/렌탈 정보를 입력하면 견적서가 자동 생성됩니다
3
내 견적 현황 — 내가 신청한 견적서의 상태를 확인합니다

3 프로필 설정 (이메일 등록)

1
2
3
4
1
이름 — 견적서에 표시되는 요청자명
2
이메일 — 견적 승인 알림을 받을 이메일 주소. 반드시 등록하세요!
3
전화번호 — 연락처 (견적서에 자동 입력)
4
저장 — 변경사항 저장
이메일을 등록해야 견적 승인 시 이메일 알림을 받을 수 있습니다. 프로필에 등록된 이메일이 견적 요청 시 자동으로 입력됩니다.

4 견적 요청

1
2
3
4
5
1
진행 단계 표시 — 기본 정보 → 기능 선택 → 상세 옵션 → 미리보기 4단계
2
견적 유형 선택 — 렌탈(기본가), 재행사(70% 할인), 판매(30% 추가) 중 선택
3
행사명 (필수) — 행사 이름을 입력합니다
4
행사일/장소 — 행사 기간과 장소 정보
5
요청자 정보 — 프로필에서 자동 입력됩니다 (이름, 이메일)
4단계를 모두 입력하면 견적이 자동 생성되어 개발팀에게 전달됩니다. 개발팀이 검토 후 승인/반려합니다.

5 내 견적 현황

1
2
3
4
1
상태 요약 카드 — 대기중, 검토중, 승인, 확정, 미진행, 반려 건수를 한눈에 확인
2
상태 필터 — 특정 상태의 견적만 필터링
3
검색 — 행사명, 요청자, 견적번호로 검색
4
상태 배지 — 각 견적의 현재 상태 표시. 클릭하면 상세 페이지로 이동
승인된 견적은 클라이언트 확인 후 "확정" 또는 "미진행" 처리가 필요합니다.

7 행사 확정 / 미진행 처리

1
2
3
4
1
행사 시작일 — 최종 확정된 행사 시작 날짜 (견적서의 행사일이 자동 입력됨)
2
행사 종료일 — 행사 종료 날짜
3
개발 마감일 — 개발팀이 준비 완료해야 하는 날짜. 행사 2주 전으로 자동 계산됩니다
4
확정 버튼 — 클릭하면 행사가 확정되고, 개발팀에게 이메일 알림이 전송됩니다
클라이언트가 행사를 진행하지 않기로 하면 "미진행" 버튼을 눌러주세요. 나중에 다시 진행하게 되면 "승인 상태로 되돌리기"로 복원할 수 있습니다.

8 행사 달력

1
2
3
4
1
내 행사 달력 — 확정된 행사만 달력에 표시됩니다
2
월 이동 — 좌우 화살표로 이전/다음 달 이동
3
범례 — 렌탈(파랑), 재행사(청록), 개발마감(빨강) 색상 구분
4
오늘 버튼 — 현재 달로 바로 이동
달력에서 이벤트를 클릭하면 견적 상세 페이지로 바로 이동합니다. 확정 처리를 하지 않은 견적은 달력에 표시되지 않습니다.

전체 워크플로우

견적 요청 → 대기중 → 검토중 → 승인 → 확정 ✓

승인 후 클라이언트 확인 → 확정(행사 진행) 또는 미진행(행사 취소) 처리
확정 시 개발팀에게 이메일 알림 + 달력에 일정 표시

⚠️ 꼭 확인해주세요

  • ✅
    견적이 승인되면 반드시 "행사 확정" 또는 "미진행" 처리를 해주세요.
    확정 처리를 해야 개발팀 달력에 일정이 등록되고, 별도 연락 없이도 개발 작업이 자동으로 진행됩니다.
  • ⚠️
    행사 일정이 변경되거나 취소되는 경우에는 개발팀에 별도로 연락해주세요.
    개발팀은 시스템에 등록된 확정 일정 기준으로만 작업을 진행합니다.
  • 📅
    개발 마감일은 행사 시작일 2주 전으로 자동 계산됩니다.
    개발팀은 이 마감일까지 준비를 완료합니다.
