## 🔒 Floor Pricing 룰

1. **Item-level floor**: 위 표의 p25를 절대 하한선으로 사용. 그 아래 견적은 의도적 예외만 허용
2. **Project-level floor**: 항목 합산 결과가 floor. 별도 카테고리 floor 두지 않음 (scope 차이로 의미 없음)
3. **예외 사유 명시**: 더 낮게 나갈 때는 ReferenceQuote.exceptionReason에 사유 기록 → baseline 산정에서 자동 제외
4. **다이어트 회피 룰**:
   - 견적서 항목 단위로 가격 분해
   - 협상 시 클라이언트가 *항목 cut*은 가능 (자동 차감)
   - *Unit price 양보*는 거부 (scope=가격 원칙)
   - "필수/선택" 마킹으로 cut 가능 항목만 사전 표시

## ⚠️ 예외 케이스 룰

다음 조건일 때만 floor 아래 가능:
- **Side + AI 자동화**: Claude/유사 도구로 인적 시간 ↓ → 일당 환산 baseline 기준 충족 시
- **R&D 투자**: 향후 재활용 가능 코드베이스 (IP 보유 전제)
- **첫 거래 신뢰 구축**: 1건만, 명시적으로 "특별가" 명목

→ ReferenceQuote에 `isException=true` + `exceptionReason` 기록. **baseline 산정에 영향 없음.**

## 🛡️ 차기 견적 시 활용 워크플로우

1. 새 견적 항목 작성 시 → `analyze-baseline.js` 실행하여 시세 확인
2. 항목별 단가가 floor 이상인지 검증
3. 견적서에 *"필수/선택"* 명시 (QuoteItem.isRequired 필드)
4. 협상 시 클라이언트 cut 요청 → 항목 단위로만 양보
5. 최종 확정되면 → ReferenceQuote에 record (isException=false) → baseline 누적
