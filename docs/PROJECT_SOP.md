# 견적 → 프로젝트 문서화 표준 SOP (v1.0)

quote-maker에서 견적 검토 후 노션에 프로젝트 페이지를 만들고 개발/배포까지 가는 **공통 워크플로우 표준**.
거의 모든 프로젝트(렌탈/행사/판매)에 일관 적용한다.

> 이 문서는 작업 환경(로컬, CI, 다른 PC, 다른 팀원)에 관계없이 동일한 절차로 진행되도록 GitHub에 보관한다.

---

## 1. 단계 책임 매트릭스

| # | 단계 | 트리거 | 담당 | 산출물 | SLA |
|---|------|--------|------|--------|-----|
| 1 | 견적 폼 입력 | 사업팀 요청 | 사업팀 | Quote (status: pending) | — |
| 2 | 견적 자동 산출 | 폼 제출 | 시스템 | 견적 초안 (pending → reviewing) | 즉시 |
| 3 | **개발팀 검토** | 검토 시작 | 개발팀 | 댓글 회신 (옵션 제시) | **영업일 1일** |
| 4 | 사업팀↔고객 합의 | 댓글 응답 | 사업팀 | A/B 옵션 선택 또는 수정 요청 | 영업일 2일 |
| 5 | **승인 게이트** | 승인 버튼 | 개발팀 | 자동 검증 통과 | 즉시 |
| 6 | **노션 페이지 자동 생성** | 승인 이벤트 | 시스템 | 메인 + 하위 5종 스켈레톤 | 즉시 |
| 7 | 설계 문서 작성 | 페이지 생성 후 | 개발 담당 | 하위 5종 본문 | D-7 까지 |
| 8 | 개발 착수 | 설계 완료 | 개발 담당 | status → 개발 진행 | — |
| 9 | 배포/현장 | 납기일 | 개발+사업 | status → 행사 진행 중 | 납기일 |
| 10 | 회수/정산 | 행사 종료 | 사업팀 | status → 완료 | 행사 +7일 |

## 3. 사전 확인 사항 표준 카테고리

### 공통 필수 (3종)
1. **디자인/콘텐츠 소스** — PSD/AI 원본 제공 주체, 폰트/로고 권한
2. **운영 프로토콜** — 장소별 인원/장비 분배, 운영 시간, 처리량
3. **추가 비용 항목** — 장비 렌탈비, 출장비, 현장 지원비, 소모품

### 도메인별 추가
- **카드프린터**: 카드 양식 / 소모품 공급 / 인프라(전원·네트워크)
- **웹/서버**: API 키 / 도메인·SSL / 인증(SSO/OIDC)
- **AI 기능**: API 사용료 / 데이터 보존 정책 / 폴백 모델

### 마지막 공통
- **계약/정산** — 계약서 시점, 계약금 40%/잔금 60% 적용 여부, 세금계산서 조건

## 4. 승인 게이트 — 자동 검증 4종 (구현 예정)

승인 버튼 누를 때 시스템이 검증, 위반 시 **모달 경고 + 무시 사유 입력 강제**.

| # | 검증 | 위반 사례 |
|---|------|-----------|
| 1 | 모든 항목 단가가 마스터 가격표 ±20% 이내 | QR 업로드 100k → 견적 150k (50% 초과) |
| 2 | 렌탈 유형이면 렌탈비 라인 존재 | 렌탈인데 개발비만 잡힘 |
| 3 | 견적 합계 = 댓글에 명시된 금액 | 댓글 90만 / 시스템 93.5만 |
| 4 | 행사일/납기일 사이 영업일 ≥ 5 | 납기 D-2 |

위 4종 모두 [QT-20260420-001 스포티파이 학생증 건](https://quote-maker-alpha.vercel.app/dashboard/quotes)에서 실제 발생한 누락/불일치 사례 기반.
